About the Curry Term Graph Visualizer

This tool can be used to visualize smaller computations of the Curry programming language as a slide show. It was created by Sascha Ecks in the context of a bachelor's thesis supervised by Prof. Michael Hanus at the CAU Kiel.

Usage

Just enter a Curry program and a main expression that should be evaluated into the corresponding fields. Note that the Curry program must define all functions, i.e., only data types but no functions from the prelude can be used. With the "term graph" option, the computation's term graphs are displayed as actual graphs so that the sharing of common subexpression is visualized.

The option "term graph as tree" displays the term graphs as trees, where shared nodes are displayed multiple times. To avoid infinitely deep tree representations of term graphs in case of graphs with circles, a maximum depth for the generated tree representations has to be specified if this option is selected.

Furthermore, the maximum number of execution steps is always limited to avoid infinite computations. The limit can be set manually but cannot exceed {{max_steps}} steps.

Note that Curry programs are compiled into the intermediate ICurry representation of programs. Thus, functions containing nested patterns in the left-hand sides of their rules are translated into simpler functions by introducing auxiliary operations which might occur during a computation.

Symbols used in the term graph

The following symbols are used in graphs:

  • Rectangle nodes represent data constructors or literals, i.e., nodes which cannot be further evaluated.
  • Choice nodes labelled with "?" represent non-deterministic choices between two expressions. Thus, non-deterministic computations are explicitly represented (in contrast to backtracking implementations). The number after the "?" is the unique choice identifier.
  • Yellow nodes identify the node to be evaluated in the next step.
  • Green nodes show an already computed result.
